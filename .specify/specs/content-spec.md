# Content Specification — Portfolio-2025

Tone & Voice
- Professional, confident, and approachable. Avoid jargon where possible; when technical depth is required, provide concise context.

Page Content Strategy
- Home: concise hero statement (one line), 1–2 supporting sentences, two CTAs (primary: view work, secondary: contact). Include a single-line tagline that summarizes expertise.
- About: short bio (3–5 paragraphs), curated highlights (impact numbers or notable clients), core skills and services offered.
- Projects: each project should follow a template (title, role, timeline, problem, approach, tech, impact/outcome, link to code/live).
- Experience: concise entries with role, company, period, a 1–2 line summary of responsibilities and outcomes.
- Contact: short instruction line, minimal form, clear privacy note and expected response time (e.g., "I usually reply within 3 business days").

SEO-conscious Copy
- Each page: unique H1; meta title and description templates.
- Project descriptions include a 1–2 sentence SEO-friendly short summary and a longer 2–4 sentence case study.

Content Templates
- Project Teaser (for gallery): Title — 1–2 sentence summary — 3 tech tags — CTA(s).
- Project Case Study (detail): Problem → Role → Solution → Impact (KPIs or qualitative results) → Tech stack → Links.

CTAs & Microcopy
- Buttons: use action-focused copy: "View Work", "Get In Touch", "See Case Study".
- Error messages: short, polite, and specific (e.g., "Please enter a valid email address").

Localization
- Primary language: English. Keep copy concise to ease future localization.

Acceptance Criteria
- All pages have required meta (title, desc) and H1 structure.
- Project templates are filled for at least 4 portfolio entries.
